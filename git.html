GIT: Sentencias más usadas

git init

Inicia un nuevo repositorio Git en el directorio actual. Esto establece el seguimiento de versiones en ese directorio y crea un subdirectorio oculto .git para almacenar la información de control de versiones.

git status

Este comando se utiliza para verificar el estado actual de tu repositorio Git. Te muestra información sobre los cambios realizados en tus archivos, los archivos que están listos para ser confirmados (staged), los archivos sin seguimiento y otros detalles sobre tu repositorio.

git config --global user.name / git config --global user.email

Este comando configura tu nombre de usuario y tu email a nivel global en Git. Se utiliza para establecer la identidad del autor de los commits en todos tus repositorios Git en tu sistema. Configurar el nombre de usuario y la dirección de correo electrónico es importante, ya que esta información se incluye en cada commit que realizas, lo que permite atribuir los cambios a las personas adecuadas.

git add

El comando git add se utiliza en Git para preparar cambios en archivos específicos o en todo el directorio de trabajo para ser incluidos en el próximo commit. Después de usar git add, los archivos o cambios se colocan en la zona de preparación (staging area), y están listos para ser confirmados con el comando git commit. Esto permite organizar los cambios y realizar confirmaciones específicas en lugar de confirmar todos los cambios en el directorio de trabajo de una vez.
